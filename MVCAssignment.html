An MVC Assignment

February 24, 2015 by Cody Alan Sanders

In this assignment, ASP.NET MVC is used to create a website for hotdog lovers everywhere! It allows users to create a profile that includes a biography and information about their hotdog experiences.

This page will explain how the different parts of the assignment- the models, views and controllers -were integrated.



Models

The first step is to create the models. A model, which is simply a class, was created for Person and Hotdog. Following the creation of the models, data annotations were used for validation. This included adding required fields, display names and ranges.



Controllers

After the models were set up and validated, Entity Framework was used to create Controllers and their associated Views. Entity Framework, essentially, created all the required code for CRUD actions.



Views

Views were created along with the controllers during the process of implementing Entity Framework. Every method in a controller has an associated view. The views display the data of their associated model classes and create the interface to interact with the data.

The views are by default a product a bootstrap, which provides inherent device responsiveness and the use of components.



While I have more experience with .NET than any other framework, and have developed web applications before, I found this assignment to be very beneficial- primarily by increasing my understanding of Entity Framework. No longer is Entity Framework just a tool I know how to apply, but I concept that I truly understand the nature. When scaffolding occurs, I have a solid idea of what is being generated and how to interact and modify the code to meet my application's needs.
